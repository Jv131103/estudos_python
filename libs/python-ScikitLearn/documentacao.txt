        Engenharia de features com Pandas e Scikit-Learn

A engenharia de features é essencial na construção de modelos de 
aprendizado de máquina robustos, e a combinação do Scikit-Learn e 
Pandas oferece uma abordagem poderosa nesse processo.

O Scikit-Learn é crucial para Ciência de Dados e Machine Learning, 
pois ele proporciona uma ampla gama de ferramentas e algoritmos de 
aprendizado de máquina. Sua biblioteca abrangente simplifica a 
implementação de modelos, oferecendo suporte desde a 
pré-processamento dos dados até a avaliação do desempenho do modelo. 
Veremos usos mais sofisticados desta biblioteca adiante.

Por enquanto, utilizaremos a parte de pré-processamento do 
Scikit-learn para transformar dados categóricos nominais e 
ordinais, bem como escalar valores numéricos.

EX:

    from sklearn.preprocessing import StandardScaler, MinMaxScaler
    from sklearn.preprocessing import LabelEncoder

    # Tratando dados categóricos - ORDINAIS
    print( df["Purchased"].unique() ) # valores únicos

    le = LabelEncoder()

    df["Purchased_encoder"] = le.fit_transform( df["Purchased"] )
    df["Grade_encoder"] = le.fit_transform( df["Grade"] )
    df

Agora vamos tratar as variáveis categóricas nominais, ou seja, 
aquelas que não possuem relação de ordem entre cada categoria. 
Neste caso usamos a técnica de One Hot Encoder, que é implementada, 
por exemplo, pelo método get_dummies() do Pandas.

EX:

    # Tratando dados categóricos - NOMINAIS
    aux = pd.get_dummies( df["Country"] )
    final = pd.concat( [df, aux], axis=1 )
    final

No entanto, claramente não precisamos de todas estas features, 
pois muitas são cópias transformadas das colunas originais. Então, 
vamos fazer outra limpeza, removendo colunas que não são mais 
necessárias. O comando a seguir pode ser feito com qualquer 
feature que julgamos irrelevante na nossa análise:

    # Filtrar colunas relevantes
    final.drop( columns=["Country", "Grade", "Purchased"] , inplace=True )
    final

Finalmente, nossas features estão completamente numéricas, o que é 
uma exigência dos modelos de Machine Learning e Deep Learning. No 
entanto, elas se apresentam em escalas bem diferentes. Nosso 
próximo comando transformará os dados por meio de Padronização e 
Normalização.

A Padronização “StandardScaler()” é mais voltada para dados que 
seguem a distribuição normal, ou seja, centrados na média. A 
Normalização “MinMaxScaler()” apenas leva a distribuição dos dados 
para outro intervalo de valores (no nosso caso, entre 10 e 20).

EX:

    # Transforma os dados com Padronização/Normalização
    sc = StandardScaler()
    final_padronizado = sc.fit_transform(final)

    mm = MinMaxScaler(feature_range=(10, 20))
    final_normalizado = mm.fit_transform(final)

Para melhor exibir os dados, criaremos novos DataFrames Pandas. 
Primeiramente, para os dados Padronizados:

    # Salvar os dados padronizados em outro dataframe
    df_final_padronizado = pd.DataFrame(final_padronizado,
    columns=final.columns)
    df_final_padronizado

Agora iremos fazer o mesmo com os dados Normalizados:

    # Salvar os dados normalizados em outro dataframe
    df_final_normalizado = pd.DataFrame(final_normalizado,
    columns=final.columns)
    df_final_normalizado

A título de curiosidade, podemos, ainda, exportar os dados salvos 
em arquivos CSV pelos seguintes comandos:

    # Exporta os dados para arquivos
    df_final_padronizado.to_csv("dados_padronizados.csv")
    df_final_normalizado.to_csv("dados_normalizados.csv")

                            Datasets

O módulo datasets fornece diversas funções para carregar conjuntos 
de dados clássicos no aprendizado de Machine Learning, além de ser 
possível gerar conjuntos de dados sintéticos para teste e validação 
de algoritmos de Aprendizado de Máquina. Esse módulo pode ser 
dividido em três partes, sendo cada uma delas adequada para 
diferentes necessidades de projeto:

    Toy Datasets: são conjuntos de dados incorporados diretamente na 
    biblioteca e podem ser facilmente carregados na memória para uso, 
    além disso, eles são pequenos e fáceis de entender, ideais para 
    aprendizado e demonstração. Alguns exemplos são:
    
        Iris Dataset (load_iris): conjunto de dados de 
        classificação para análise de flores.
        
        Diabetes Dataset (load_diabetes): conjunto de dados de 
        regressão para prever a progressão da diabetes.

O carregamento do conjunto de dados é simples, após importar do 
conjunto de dados, ele retorna um objeto Bunch, que é similar a um 
dicionário Python. Esse objeto contém chaves como data, target, 
feature_names e DESCR, facilitando o acesso às informações do 
dataset, como demonstra o código a seguir:

    from sklearn.datasets import load_iris

    # Carrega o dataset
    iris = load_iris()

    # Acessar dados e rótulos
    X, y, name = iris.data, iris.target, iris.feature_names

    print("Características:\n", X[:5])
    print("Rótulos:\n", y[:5])
    print("Nome das características:\n", name)

    # Acessar metadados
    print("Metadados:\n", iris.DESCR)

O resultado esperado é que seja importado o dataset Iris, e também 
que ele exiba os dados, rótulos e características do conjunto de 
dados, como mostra a figura “Resultado importe Iris”.

    Geradores de conjuntos de dados sintéticos: é possível criar 
    dados artificiais que seguem distribuições específicas, ou seja, 
    permitem controlar características estatísticas dos dados, como 
    correlação e relevância das features. Esses dados são úteis para 
    realizar testes controlados, como, por exemplo, o impacto da 
    escala do dataset (número de amostras e features) em modelos de 
    Machine Learning. Alguns exemplos são:
    
        make_classification: gera conjuntos de dados para problemas 
        de classificação, onde é possível especificar o número de 
        amostras, características informativas, características 
        redundantes e clusters.
        
        make_regression: gera conjuntos de dados para problemas de 
        regressão, com controle sobre o número de amostras, features 
        e nível de ruído.
        
        make_blobs: gera clusters de dados gaussianos para tarefas 
        de clusterização.
        
        make_moons e make_circles: gera dados para problemas de 
        classificação não lineares.

                Pré processamento

O módulo preprocessing é utilizado para manipular e transformar 
dados antes de aplicá-los a algoritmos de aprendizado de máquina. 
Entre as diversas possibilidades de pré-processamento que a 
biblioteca disponibiliza, estão as técnicas de escalonamento de 
características, imputação de valores ausentes, codificação de 
variáveis categóricas e normalização de dados. Iremos utilizar um 
conjunto de dados real para demonstrar como aplicar as principais 
técnicas de pré-processamento disponíveis no scikit-learn. Isso 
ajudará a entender na prática como essas transformações afetam os 
dados e por que são importantes.

O dataset Wine do sklearn é um conjunto de dados que contém 
informações de diferentes vinhos, tais como teor alcoólico, acidez, 
magnésio, fenóis, entre outras.

    from sklearn.datasets import load_wine
    import pandas as pd
    import numpy as np

    # Carregando o dataset
    wine = load_wine()
    X = wine.data  # Características
    y = wine.target  # Rótulos

    # Convertendo para DataFrame para melhor visualização
    df = pd.DataFrame(X, columns=wine.feature_names)
    df['target'] = y
    df.head()

Na figura “Exibe df wine”, conseguimos visualizar apenas alguns dos 
atributos, note que existem escalas de valores diferentes.

    Escalonamento e normalização: em muitos casos, as variáveis dos 
    conjuntos de dados possuem diferentes escalas, o que pode 
    impactar negativamente a eficácia do modelo de ML. O dataset 
    Wine tem essa característica com variáveis em escalas diferentes 
    (algumas variam entre 0 e 10, outras entre 0 e 1000). Isso pode 
    causar problemas para algoritmos que são sensíveis à escala, 
    como SVM e KNN. Vamos aplicar dois métodos de escalonamento:
    
        StandardScaler: transforma os dados para que possuem média 
        0 e desvio padrão 1. Utilizado quando os dados seguem uma 
        distribuição aproximadamente normal.
        
        MinMaxScaler: transforma os dados para que fiquem dentro de 
        um intervalo específico, geralmente [0, 1]. Utilizado em 
        algoritmos baseados em distância.

    EX:

        from sklearn.preprocessing import StandardScaler, MinMaxScaler
        import pandas as pd

        # Aplicando StandardScaler
        scaler_standard = StandardScaler()
        X_scaled = scaler_standard.fit_transform(X)

        # Aplicando MinMaxScaler
        scaler_minmax = MinMaxScaler()
        X_minmax_scaled = scaler_minmax.fit_transform(X)

        # Mostrando as primeiras linhas dos dados escalados
        df_scaled = pd.DataFrame(X_scaled, columns=wine.feature_names)
        df_scaled.head()

        df_minmax = pd.DataFrame(X_minmax_scaled, columns=wine.feature_names)
        df_minmax.head()

A figura “Resultado normalização” exibe os resultados dos dois 
métodos de escalonamento. No primeiro bloco de dados, após a 
aplicação do StandardScaler, as variáveis escaladas possuem média 
0 e desvio padrão 1, o que é importante para modelos que assumem 
uma distribuição gaussiana ou precisam de dados centralizados. No 
segundo bloco de dados, as colunas escaladas pelo MinMaxScaler 
foram normalizadas para o intervalo [0, 1]. Aplicado em algoritmos 
que utilizam distâncias ou gradientes, garantindo que as features 
estejam na mesma escala, sem permitir que uma tenha influência 
indevida no modelo por conta de sua magnitude original.

    Imputação de valores ausentes: quando lidamos com dados reais, 
    muitas vezes encontramos valores ausentes. Esses valores 
    precisam ser tratados para que o modelo de Machine Learning 
    funcione corretamente. O sklearn.preprocessing oferece duas 
    abordagens principais para imputar (preencher) valores ausentes:
    
        SimpleImputer: substitui os valores ausentes por uma 
        estratégia estatística simples, como a média, mediana, 
        moda, ou um valor constante fornecido. É um algoritmo 
        rápido e barato computacionalmente, porém, ao utilizá-lo 
        para substituir valores ausentes por uma média ou mediana, 
        esse algoritmo pode ignorar correlações ou padrões 
        complexos nos dados.
        
        KNNImputer: utiliza o algoritmo de K-Nearest Neighbors 
        (KNN) para estimar valores ausentes com base em valores 
        vizinhos no conjunto de dados. Essa técnica faz a imputação 
        com base em padrões de correlação entre as variáveis, por 
        isso é computacionalmente mais cara e pode ser sensível ao 
        valor de k. Além disso, pode ser ineficaz se não houver 
        correlação entre as variáveis ou se houver muitas dimensões 
        com valores ausentes.

Codificação de variáveis categóricas: em projetos de Ciência de 
Dados e Aprendizado de Máquina, nos deparamos com variáveis 
categóricas, ou seja, que assumem valores discretos ou qualitativos, 
como “gênero”, “cor”, “categoria de produto”, entre outros. De 
forma geral, os algoritmos de Machine Learning esperam que as 
entradas sejam numéricas. Logo, essas variáveis precisam ser 
transformadas antes que possam ser utilizadas como input para os 
modelos. Esse processo é conhecido como codificação de variáveis 
categóricas. No entanto, a maneira como essas variáveis são 
codificadas pode afetar o desempenho do modelo, especialmente em 
termos de eficiência e capacidade de generalização. Por essa razão, 
existem diferentes formas de transformar variáveis categóricas em 
valores numéricos. Os métodos mais comuns e utilizados com o 
scikit-learn são:

    LabelEncoder: converte cada categoria em um valor inteiro único. 
    Cada categoria recebe um número inteiro diferente, e os valores 
    numéricos não possuem relação direta entre si além de representar 
    categorias distintas. As categorias são ordenadas alfabeticamente 
    e cada categoria recebe um valor correspondente.
    
    OneHotEncoder: cria uma nova coluna binária (0 ou 1) para cada 
    categoria, sem considerar qualquer hierarquia entre elas. Essa 
    técnica é especialmente útil quando não há relação ordinal entre 
    as categorias.
    
    OrdinalEncoding: atribui valores inteiros às categorias, mas 
    preserva uma relação de ordem intrínseca entre elas. Este método 
    é apropriado quando as categorias possuem uma ordem natural (por 
    exemplo, “pequeno”, “médio”, “grande”). A ordem das categorias 
    deve ser explicitamente definida durante o processo de 
    codificação, garantindo que os valores numéricos respeitem a 
    hierarquia natural dos dados.

                Módulo model_selection

O módulo model_selection oferece uma série de ferramentas e 
utilitários para a construção de pipelines robustos de Aprendizado
de Máquina. Entre as funções mais importantes, destacamos a 
divisão de dados, validação cruzada e ajuste de hiperparâmetros:

    . Divisão de conjuntos de dados: em qualquer projeto de 
    Aprendizado de Máquina, uma prática essencial é separar os 
    dados em conjuntos de treinamento e teste. Isso permite 
    treinar o modelo em um subconjunto dos dados e avaliá-lo em 
    outro subconjunto, garantindo que o modelo não seja avaliado 
    nos mesmos dados em que foi treinado.

A função train_test_split facilita esse processo, dividindo os 
dados em duas ou mais partes de forma aleatória ou estratificada. 
O parâmetro test_size define a proporção dos dados reservada para 
o teste, e o parâmetro random_state garante a reprodutibilidade do 
processo.

EX:

    import pandas as pd
    from sklearn.datasets import load_wine
    from sklearn.model_selection import train_test_split

    # Carregando novamente o dataset
    wine = load_wine()
    X = wine.data  # Características
    y = wine.target  # Rótulos

    # Convertendo para DataFrame para melhor visualização
    df = pd.DataFrame(X, columns=wine.feature_names)
    df['target'] = y

    # Escolhendo X e y para fazer o split
    X = df.drop('target', axis=1)
    y = df['target']

    # Dividindo o DataFrame em 70% para treino e 30% para teste
    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

    # Conjuntos de dados
    print(f'Tamanho do conjunto de dados original: {X.shape}')
    print(f'Tamanho do conjunto de treinamento: {X_train.shape}')
    print(f'Tamanho do conjunto de teste: {X_test.shape}')

    # Exibir as primeiras linhas do conjunto antes e depois do split
    print("Conjunto de dados antes do split:")
    display(df['target'].head())
    print("\nConjunto de treinamento:")
    display(y_train.head())

Realizamos a divisão do dataset na proporção de 70% treino e 30% 
teste, para garantir reprodutibilidade da divisão foi utilizado o 
valor 42 como semente. Isso assegura que, quando o mesmo valor de 
random_state é utilizado, a divisão será a mesma a cada execução. 
Note que o conjunto de treinamento após o train_test_split possui 
uma ordem aleatória diferente do conjunto de dados original. A 
figura “Aplicação train_test_split” exibe o resultado desse 
processo.

Dica:
    - Validação cruzada (cross_val_score e KFold), que ajudam a 
    avaliar a robustez do modelo.
    
    - Busca de hiperparâmetros (GridSearchCV e RandomizedSearchCV) 
    permitem otimizar hiperparâmetros de modelos para melhor 
    desempenho.

            Estimadores e treinamento de modelos

No scikit-learn, o termo “estimators” (ou estimadores) são modelos 
ou algoritmos que podem aprender a partir de dados (treinamento) e 
fazer previsões (predição), como ilustra a figura “Utilização dos 
métodos fit predict”. Em outras palavras, um estimador é qualquer 
objeto que implemente os métodos fit() e predict(), que são:

    fit(X, y): treina o modelo (estimador) a partir dos dados de 
    entrada X e dos rótulos y.
    
    predict(X): realiza previsões com base nos dados de entrada X 
    após o treinamento.

Os estimadores são a base de qualquer algoritmo de aprendizado 
supervisionado ou não supervisionado. E podem ser divididos em 
várias categorias:

    Modelos de Regressão: como LinearRegression e Ridge.
    
    Modelos de Classificação: como LogisticRegression, SVC 
    (Support Vector Classifier) e RandomForestClassifier.
    
    Algoritmos de Clusterização: como KMeans e DBSCAN.
    
    Redução de Dimensionalidade: como PCA (Principal Component 
    Analysis).

O método score() é utilizado no scikit-learn para avaliar o 
desempenho de um modelo, mas o comportamento desse método varia 
de acordo com o tipo de modelo que está sendo utilizado 
(classificação, regressão, clusterização, entre outros). Veja a 
figura “Utilização dos métodos score”.

    Problema de classificação: o método score() retorna a acurácia 
    do modelo, que é a proporção de previsões corretas.
    
    Problema de regressão: retorna o coeficiente de determinação 
    (R²), que mede o quão bem os valores previstos se ajustam aos 
    valores reais.

Estimadores de classificação: são utilizados para problemas onde o 
objetivo é prever rótulos discretos ou categorias. Os mais 
conhecidos são:

    sklearn.linear_model: Regressão Logística (LogisticRegression) 
    para problemas de classificação binária ou multiclasse.
    
    sklearn.ensemble: RandomForestClassifier, AdaBoostClassifier, 
    e outros métodos de ensemble.
    
    sklearn.svm: máquinas de vetores de suporte (SVM) para 
    classificação.
    
    sklearn.neighbors: classificação com K-Nearest Neighbors (KNN).

Vamos utilizar o dataset wine para treinar um classificador de 
vinhos com base em suas características. Primeiro, iremos carregar 
e preparar os dados para serem utilizados pelo estimador.

EX:

    import pandas as pd
    from sklearn.datasets import load_wine
    from sklearn.model_selection import train_test_split

    # Carregando novamente o dataset
    wine = load_wine()
    X = wine.data  # Características
    y = wine.target  # Rótulos

    # Convertendo para DataFrame para melhor visualização
    df = pd.DataFrame(X, columns=wine.feature_names)
    df['target'] = y

    # Escolhendo X e y para fazer o split
    X = df.drop('target', axis=1)
    y = df['target']

    # Dividindo o DataFrame em 70% para treino e 30% para teste
    X_train, X_test, y_train, y_test = train_test_split(X, y, test_size=0.3, random_state=42)

Com o conjunto de dados pronto, podemos seguir para a etapa de 
treinamento. Nesta etapa, iremos utilizar um RandomForestClassifier 
(mas poderia ser qualquer outro algoritmo de classificação, pois a 
forma de utilização não muda). A função fit(), recebe X_train e 
y_train como entradas para o treinamento do modelo. Após rodar o 
fit() o modelo estará treinado. Para realizar uma predição, devemos 
chamar o método predict() utilizando uma entrada com o mesmo número 
de atributos das amostras alocadas em X_tain, neste caso, vamos usar 
X_test que retorna uma predição para cada entrada. O método score(), 
por sua vez, compara os resultados preditos versus os rótulos reais, 
e calcula a acurácia do modelo treinado.

EX:

    from sklearn.ensemble import RandomForestClassifier

    # Instanciando o modelo
    model = RandomForestClassifier()

    # Treinamento do modelo
    model.fit(X_train, y_train)

    # Predição
    y_pred = model.predict(X_test)

    # Avaliando a acurácia do modelo usando o método score()
    accuracy = model.score(X_test, y_test)
    print(f"Acurácia do modelo: {accuracy:.2f}")

Uma das grandes vantagens do scikit-learn é a sua interface 
unificada para todos os estimadores. Isso significa que, uma vez 
que você tenha organizado seus dados e feito a divisão entre os 
conjuntos de treino e teste, você pode facilmente trocar entre 
diferentes modelos (estimadores) com apenas pequenas alterações no 
código. Os métodos principais, como fit(), predict() e score(), 
estão presentes em praticamente todos os estimadores no 
scikit-learn, independentemente de serem algoritmos de 
classificação, regressão ou clusterização. Isso torna o processo de 
experimentação e comparação entre diferentes modelos extremamente 
simples.

EX:

    from sklearn.ensemble import RandomForestClassifier
    from sklearn.svm import SVC
    from sklearn.neighbors import KNeighborsClassifier

    # Testando diferentes algoritmos de classificação

    # Random Forest
    model_rf = RandomForestClassifier()
    model_rf.fit(X_train, y_train)
    accuracy_rf = model_rf.score(X_test, y_test)
    print(f"Acurácia com Random Forest: {accuracy_rf:.2f}")

    # SVM
    model_svm = SVC()
    model_svm.fit(X_train, y_train)
    accuracy_svm = model_svm.score(X_test, y_test)
    print(f"Acurácia com SVM: {accuracy_svm:.2f}")

    # KNN
    model_knn = KNeighborsClassifier()
    model_knn.fit(X_train, y_train)
    accuracy_knn = model_knn.score(X_test, y_test)
    print(f"Acurácia com KNN: {accuracy_knn:.2f}")

Estimadores de regressão: são utilizados para prever valores

            Ferramentas para avaliação e otimização

A avaliação de modelos é uma etapa crucial em qualquer pipeline de 
Machine Learning. Dependendo do tipo de tarefa (classificação, 
regressão ou clusterização), diferentes métricas são utilizadas 
para medir o quão bem o modelo está desempenhando. A seguir, 
exploraremos as principais métricas oferecidas pelo scikit-learn.

Acurácia ou taxa de acerto é a métrica mais utilizada em problemas 
de classificação, pois mede o número de acertos do modelo dividido 
pelo número total testado. Por exemplo, se o modelo possui uma 
acurácia de 0.95 (95%), o seu modelo acerta 95 de 100 previsões. 
Essa é a métrica mais simples e mais utilizada em datasets onde as 
classes são balanceadas, ou seja, possuímos proporções parecidas de 
amostras para cada uma das categorias.

EX:

    import numpy as np
    from sklearn.metrics import accuracy_score # importa método da acurácia

    # Cria dois array de teste
    y_pred = [0, 2, 1, 3]
    y_true = [0, 1, 2, 3]

    # Calcula acurácia
    accuracy_score(y_true, y_pred)

    # Resultado
    # 0.5

Dependendo do problema, pode ser necessário utilizar outras 
métricas de avaliação do modelo de classificação. A matriz de 
confusão exibe de forma simples os resultados de classificação 
onde as linhas representam os valores reais e as colunas os 
valores preditos, a diagonal principal é composta pelos acertos 
do modelo e os demais valores são os erros cometidos como pode ser 
observado na figura “Matriz de confusão”.

Da matriz de confusão podemos extrair as seguintes análises:

    Verdadeiro Positivo (VP): quando o modelo prediz que é 
    positivo e realmente é positivo.
    
    Verdadeiro Negativo (VN): quando o modelo prediz que é 
    negativo realmente é negativo.
    
    Falso Positivo (FP): quando o modelo prediz que é positivo, 
    mas na verdade é negativo.
    
    Falso Negativo (FN): quando o modelo prediz que é negativo, 
    mas na verdade é positivo.

A partir da matriz de confusão extraímos diversas medidas de 
performance, tais como: F1-Score, Recall, Precision e 
Especificidade. Iremos utilizar o dataset wines treinado com KNN 
para visualizar essas métricas de performance.

Após treinar o modelo, vamos utilizar as métricas de performance. 
Observe que cada métrica tem um papel específico, e a matriz de 
confusão oferece uma visão detalhada do desempenho do modelo.

EX:

    # Importando todas as métricas necessárias
    from sklearn.metrics import accuracy_score, precision_score, recall_score, f1_score, confusion_matrix

    # Calculando e exibindo a Acurácia
    accuracy = accuracy_score(y_test, y_pred)
    print(f"Acurácia: {accuracy:.2f}")

    # Calculando e exibindo a Precisão
    precision = precision_score(y_test, y_pred, average='weighted')
    print(f"Precisão (Weighted): {precision:.2f}")

    # Calculando e exibindo o Recall
    recall = recall_score(y_test, y_pred, average='weighted')
    print(f"Recall (Weighted): {recall:.2f}")

    # Calculando e exibindo o F1-Score
    f1 = f1_score(y_test, y_pred, average='weighted')
    print(f"F1-Score (Weighted): {f1:.2f}")

    # Calculando e exibindo a Matriz de Confusão
    matrix = confusion_matrix(y_test, y_pred)
    print("Matriz de Confusão:")
    print(matrix)
